PROJECTS

Automated ships steam boiler

What is a steam boiler: Essentially, a boiler is a container into which water can be fed and by the application of heat is evaporated continuously into steam. Steam is raised mainly for heating the heavy fuel oil used to fire the ‘diesel engines’ , for space heating , to power cargo handling pumps and as an aid to cleaning storage tanks and surfaces. This program controls the ignition sequence of the burner in order to create steam. Along with the boiler program a simulator is included to simulate loads and to test the PID regulation. Showcase video (v2.0) Here Goal of this program The idea behind of this project is to provide a support project for my bachelor degree paper "Automation of ships steam boiler". When the project is finished the bachelor degree will be accompanied by the complete PLC program and a simulator. Second intention of the project is to create a reference project that presents my automation skills and as such this project will never be really finished because there is always something to improve. The project can be downloaded from GitHub from here.

TKAutomationLib

Codesys library that supports various project of neighbour repositories Version: 1.0 Contents of the library Pump control logic Pump model for simulation Tank control logic Tank model with several inputs for inflow and outflow Workplace with color changing functionality, tabs, navigation and status bar with automatic time reading How to use the library Control logic and simulation model objects: These objects are basically used by calling it using the Input Assistant in your project. Connect the variables to the IOs and compile the program to the PLC. For the visualisations drag and drop the corresponding control objects visualisation into your project and connect the correct variables to the control object. Model objects for simulation should be run on a separate PLC, OPC server or atleast separate PRG. Simulation communication should be done only by IOs. Workplace WorkplaceHandler function needs to be called in a PRG. Variable arrays need to be created and function block needs to be parametrized with project specific values. Visualization is of size 1980x1024 and should be drag and dropped to an empty visualisation. In the outputs of the WorkplaceHandler there is an array variable in which the first bit dictates which tab should be visible. All project visualisations should have that bit in its "Visible" parameter. Color can be set for the navigation bar, buttons and the status bar by entering hex code RGB when parametrizing WorkplaceHandler function in the format of 16#FFRRGGBB. The library is intended to be installed into the Codesys enviroment but it is protected from editing by a password. The library can be downloaded from GitHub from here.

PLC Programming school

Information about Structured Text programming is scarce to say the least so I have decided to solve this problem and dedicate a online school to teaching ST Code to anyone who wants to learn it. The school is in progress of being built as you are reading this text. It will focus on the IEC61311-3 standard and on Structured Text programming for PLCs. ST Code course will start as a introduction to what it really is and how to use it but after the introduction we will focuse on real life industry problems an we will solve them using structured text programming, functions, simulators and much more.
